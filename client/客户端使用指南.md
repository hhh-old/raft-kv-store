# Raft KV Store 客户端使用指南

## 一、客户端概述

Raft KV Client 是一个功能完善的 Java 客户端，专为 Raft KV Store 设计。

### 核心特性

✅ **幂等性支持**：自动生成或使用客户端提供的 requestId  
✅ **自动重试**：超时或失败时自动重试  
✅ **Leader 自动重定向**：自动跟随 Leader 切换  
✅ **指数退避**：避免雪崩效应  
✅ **健康检查**：检测服务可用性  
✅ **线程安全**：可在多线程环境中使用

## 二、快速开始

### 2.1 添加依赖

在你的项目 `pom.xml` 中添加：

```xml
<dependency>
    <groupId>com.raftkv</groupId>
    <artifactId>raft-kv-client</artifactId>
    <version>1.0.0</version>
</dependency>
```

### 2.2 创建客户端

```java
import com.raftkv.client.RaftKVClient;
import com.raftkv.client.entity.KVResponse;

// 创建客户端
RaftKVClient client = RaftKVClient.builder()
        .serverUrls(Arrays.asList(
                "http://localhost:9081",
                "http://localhost:9082",
                "http://localhost:9083"
        ))
        .maxRetries(3)              // 最大重试次数
        .timeoutSeconds(3)          // 超时时间（秒）
        .build();
```

### 2.3 基本操作

```java
// PUT 操作
KVResponse putResponse = client.put("name", "Alice");
System.out.println("成功: " + putResponse.isSuccess());
System.out.println("RequestId: " + putResponse.getRequestId());

// GET 操作
KVResponse getResponse = client.get("name");
System.out.println("name = " + getResponse.getValue());

// DELETE 操作
KVResponse deleteResponse = client.delete("name");
System.out.println("删除成功: " + deleteResponse.isSuccess());
```

## 三、高级功能

### 3.1 幂等性支持

**方式 1：自动生成 requestId**

```java
// 客户端自动生成 requestId
KVResponse response = client.put("name", "Alice");
String requestId = response.getRequestId();  // 获取生成的 requestId
```

**方式 2：指定 requestId（推荐）**

```java
// 基于业务逻辑生成 requestId
String requestId = "order-12345-payment";

// 首次请求
KVResponse response1 = client.put("status", "paid", requestId);

// 如果超时或失败，重试时使用相同 requestId
KVResponse response2 = client.put("status", "paid", requestId);

// 服务端保证幂等：相同 requestId 只执行一次
```

### 3.2 自动重试机制

客户端内置自动重试，无需手动处理：

```java
// 配置重试参数
RaftKVClient client = RaftKVClient.builder()
        .serverUrls(Arrays.asList("http://localhost:9081"))
        .maxRetries(3)              // 最多重试 3 次
        .timeoutSeconds(3)          // 每次超时 3 秒
        .build();

// 如果请求失败，客户端会自动重试
KVResponse response = client.put("key", "value");
```

**重试策略**：
- 第 1 次失败：等待 100ms 后重试
- 第 2 次失败：等待 200ms 后重试
- 第 3 次失败：等待 400ms 后重试
- 第 4 次失败：抛出异常

### 3.3 Leader 自动重定向

当请求发送到 Follower 时，客户端自动重定向到 Leader：

```java
// 假设当前连接的节点是 Follower
KVResponse response = client.put("key", "value");

// 客户端收到 NOT_LEADER 响应
// 自动获取 Leader 地址
// 自动重定向到 Leader
// 对应用层透明
```

**手动更新 Leader**：

```java
// 如果你知道新的 Leader 地址
client.updateLeader("http://localhost:9082");
```

### 3.4 健康检查

```java
// 检查服务是否可用
boolean healthy = client.healthCheck();
if (healthy) {
    System.out.println("服务正常");
} else {
    System.out.println("服务不可用");
}
```

### 3.5 获取集群统计信息

```java
String stats = client.getStats();
System.out.println(stats);

// 输出示例：
// {
//   "role": "LEADER",
//   "endpoint": "127.0.0.1:8081",
//   "currentTerm": 5,
//   "commitIndex": 100,
//   "lastApplied": 100,
//   "keyCount": 50
// }
```

## 四、配置选项

### 4.1 Builder 参数

```java
RaftKVClient client = RaftKVClient.builder()
        // 必填：服务器地址列表
        .serverUrls(Arrays.asList(
                "http://localhost:9081",
                "http://localhost:9082",
                "http://localhost:9083"
        ))
        
        // 可选：最大重试次数（默认 3）
        .maxRetries(5)
        
        // 可选：超时时间（秒，默认 3）
        .timeoutSeconds(5)
        
        .build();
```

### 4.2 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| serverUrls | List<String> | 必填 | 服务器地址列表 |
| maxRetries | int | 3 | 最大重试次数 |
| timeoutSeconds | int | 3 | 请求超时时间（秒） |

## 五、使用场景

### 5.1 场景 1：简单 KV 操作

```java
RaftKVClient client = RaftKVClient.builder()
        .serverUrls(Arrays.asList("http://localhost:9081"))
        .build();

// 写入配置
client.put("db.host", "localhost");
client.put("db.port", "5432");

// 读取配置
String host = client.get("db.host").getValue();
String port = client.get("db.port").getValue();
```

### 5.2 场景 2：订单处理（幂等性）

```java
public class OrderService {
    private RaftKVClient client;
    
    public void processOrder(String orderId, String status) {
        // 生成幂等 requestId
        String requestId = "order-" + orderId + "-status";
        
        // 更新订单状态（支持重试）
        KVResponse response = client.put(
            "order:" + orderId + ":status",
            status,
            requestId
        );
        
        if (!response.isSuccess()) {
            throw new RuntimeException("Failed to update order status");
        }
    }
}
```

### 5.3 场景 3：计数器（并发安全）

```java
public class CounterService {
    private RaftKVClient client;
    
    public void increment(String counterName) {
        // 获取当前值
        KVResponse response = client.get("counter:" + counterName);
        int current = response.getValue() != null ? 
                Integer.parseInt(response.getValue()) : 0;
        
        // 更新值（使用幂等 requestId）
        String requestId = "counter-" + counterName + "-" + System.currentTimeMillis();
        client.put("counter:" + counterName, String.valueOf(current + 1), requestId);
    }
}
```

### 5.4 场景 4：会话管理

```java
public class SessionManager {
    private RaftKVClient client;
    
    public void createSession(String sessionId, String userId) {
        String key = "session:" + sessionId;
        String value = userId;
        String requestId = "session-create-" + sessionId;
        
        client.put(key, value, requestId);
    }
    
    public String getUserId(String sessionId) {
        KVResponse response = client.get("session:" + sessionId);
        return response.getValue();
    }
    
    public void deleteSession(String sessionId) {
        String requestId = "session-delete-" + sessionId;
        client.delete("session:" + sessionId, requestId);
    }
}
```

## 六、最佳实践

### 6.1 RequestId 生成策略

**推荐策略**：

```java
// 策略 1：基于业务键（推荐）
String requestId = "order-" + orderId + "-payment";

// 策略 2：基于操作类型 + 时间戳
String requestId = "put-" + key + "-" + System.currentTimeMillis();

// 策略 3：UUID（简单，但重试时需保存）
String requestId = UUID.randomUUID().toString();
```

**不推荐**：

```java
// ❌ 每次生成不同的 requestId（无法幂等）
String requestId = UUID.randomUUID().toString();
client.put(key, value, requestId);  // 重试时会生成新的 requestId
```

### 6.2 错误处理

```java
try {
    KVResponse response = client.put("key", "value");
    
    if (response.isSuccess()) {
        // 成功处理
        System.out.println("操作成功");
    } else {
        // 业务错误
        System.err.println("操作失败: " + response.getError());
    }
} catch (RuntimeException e) {
    // 网络错误或超过重试次数
    System.err.println("请求失败: " + e.getMessage());
}
```

### 6.3 性能优化

```java
// 1. 复用客户端实例（线程安全）
RaftKVClient client = RaftKVClient.builder()
        .serverUrls(Arrays.asList("http://localhost:9081"))
        .build();

// 2. 批量操作时共享 requestId 前缀
String batchId = "batch-" + System.currentTimeMillis();
for (int i = 0; i < 100; i++) {
    String requestId = batchId + "-" + i;
    client.put("key-" + i, "value-" + i, requestId);
}

// 3. 合理设置超时和重试
RaftKVClient client = RaftKVClient.builder()
        .serverUrls(Arrays.asList("http://localhost:9081"))
        .maxRetries(3)          // 不要设置太大
        .timeoutSeconds(3)      // 根据网络情况调整
        .build();
```

## 八、常见问题

### Q1: 如何保证幂等性？

**A**: 客户端生成 requestId，服务端基于 requestId 去重。重试时使用相同 requestId 即可。

```java
String requestId = "order-123-payment";
client.put("order:123:status", "paid", requestId);  // 重试时用相同 requestId
```

### Q2: Leader 切换时怎么办？

**A**: 客户端自动处理。收到 NOT_LEADER 响应后自动重定向到新 Leader。

### Q3: 超时后是否安全重试？

**A**: 安全。只要使用相同 requestId，服务端会返回缓存结果，不会重复执行。

### Q4: 客户端是否线程安全？

**A**: 是的。RaftKVClient 是线程安全的，可以在多线程环境中共享。

### Q5: 如何选择 serverUrls？

**A**: 建议填写所有节点地址。客户端会自动找到 Leader。

```java
.serverUrls(Arrays.asList(
    "http://localhost:9081",
    "http://localhost:9082",
    "http://localhost:9083"
))
```

## 九、故障排查

### 9.1 连接失败

**症状**：`ConnectException: Connection refused`

**原因**：服务未启动或地址错误

**解决**：
```bash
# 检查服务是否运行
curl http://localhost:9081/kv/health
```

### 9.2 超时错误

**症状**：`HttpTimeoutException: request timed out`

**原因**：网络慢或集群繁忙

**解决**：
```java
// 增加超时时间
RaftKVClient client = RaftKVClient.builder()
        .timeoutSeconds(10)  // 增加到 10 秒
        .build();
```

### 9.3 超过重试次数

**症状**：`RuntimeException: PUT failed after 3 attempts`

**原因**：集群不可用或网络问题

**解决**：
1. 检查集群状态：`curl http://localhost:9081/kv/stats`
2. 检查网络连接
3. 增加重试次数：`.maxRetries(5)`

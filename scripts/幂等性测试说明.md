# 幂等性测试脚本

## 一、测试说明

本脚本用于测试 Raft KV Store 的幂等性实现。

### 测试场景

1. **正常 PUT 操作**：首次写入
2. **重复 PUT 操作**：使用相同 requestId 重试
3. **不同 requestId 的 PUT**：验证是否视为不同请求
4. **超时重试**：模拟超时后重试
5. **DELETE 幂等**：删除操作的幂等性

### 测试 7：模拟超时重试

```bash
# 第一次请求（假设超时）
curl -X PUT http://localhost:9081/kv/email \
  -H "Content-Type: application/json" \
  -d '{
    "value": "alice@example.com",
    "requestId": "req-timeout-001"
  }'

# 假设超时了，客户端重试（使用相同 requestId）
curl -X PUT http://localhost:9081/kv/email \
  -H "Content-Type: application/json" \
  -d '{
    "value": "alice@example.com",
    "requestId": "req-timeout-001"
  }'

# 预期：
# 1. 如果第一次实际成功了，第二次直接返回缓存结果
# 2. 如果第一次失败了，第二次正常执行
# 3. 无论如何，email 的值只写入一次
```

## 四、自动化测试脚本（PowerShell）

```powershell
# test-idempotency.ps1

$baseUrl = "http://localhost:9081"

Write-Host "=== 幂等性测试 ===" -ForegroundColor Green

# 测试 1：正常 PUT
Write-Host "`n[测试 1] 正常 PUT 操作" -ForegroundColor Yellow
$response1 = Invoke-RestMethod -Uri "$baseUrl/kv/name" -Method Put `
    -Body '{"value": "Alice"}' -ContentType "application/json"
Write-Host "响应: $($response1 | ConvertTo-Json -Compress)"

# 测试 2：带 requestId 的 PUT
Write-Host "`n[测试 2] 带 requestId 的 PUT" -ForegroundColor Yellow
$response2 = Invoke-RestMethod -Uri "$baseUrl/kv/age" -Method Put `
    -Body '{"value": "25", "requestId": "req-001"}' -ContentType "application/json"
Write-Host "响应: $($response2 | ConvertTo-Json -Compress)"

# 测试 3：重复请求（幂等）
Write-Host "`n[测试 3] 重复请求（幂等性测试）" -ForegroundColor Yellow
$response3 = Invoke-RestMethod -Uri "$baseUrl/kv/age" -Method Put `
    -Body '{"value": "25", "requestId": "req-001"}' -ContentType "application/json"
Write-Host "响应: $($response3 | ConvertTo-Json -Compress)"

if ($response3.requestId -eq "req-001" -and $response3.value -eq "25") {
    Write-Host "✅ 幂等性测试通过！" -ForegroundColor Green
} else {
    Write-Host "❌ 幂等性测试失败！" -ForegroundColor Red
}

# 测试 4：不同 requestId
Write-Host "`n[测试 4] 不同 requestId（新请求）" -ForegroundColor Yellow
$response4 = Invoke-RestMethod -Uri "$baseUrl/kv/age" -Method Put `
    -Body '{"value": "26", "requestId": "req-002"}' -ContentType "application/json"
Write-Host "响应: $($response4 | ConvertTo-Json -Compress)"

# 测试 5：读取验证
Write-Host "`n[测试 5] 读取数据验证" -ForegroundColor Yellow
$response5 = Invoke-RestMethod -Uri "$baseUrl/kv/age" -Method Get
Write-Host "响应: $($response5 | ConvertTo-Json -Compress)"

if ($response5.value -eq "26") {
    Write-Host "✅ 数据更新正确！" -ForegroundColor Green
} else {
    Write-Host "❌ 数据更新失败！" -ForegroundColor Red
}

# 测试 6：DELETE 幂等
Write-Host "`n[测试 6] DELETE 幂等性" -ForegroundColor Yellow
$response6a = Invoke-RestMethod -Uri "$baseUrl/kv/name?requestId=del-001" -Method Delete
Write-Host "第一次删除: $($response6a | ConvertTo-Json -Compress)"

$response6b = Invoke-RestMethod -Uri "$baseUrl/kv/name?requestId=del-001" -Method Delete
Write-Host "第二次删除: $($response6b | ConvertTo-Json -Compress)"

if ($response6a.requestId -eq $response6b.requestId) {
    Write-Host "✅ DELETE 幂等性测试通过！" -ForegroundColor Green
} else {
    Write-Host "❌ DELETE 幂等性测试失败！" -ForegroundColor Red
}

Write-Host "`n=== 测试完成 ===" -ForegroundColor Green
```

运行测试：

```powershell
cd d:\project\Claudecode\raft-kv-store
.\scripts\test-idempotency.ps1
```

## 六、客户端最佳实践

### 6.1 生成 requestId 的策略

```java
// 策略 1：基于业务键（推荐）
String requestId = "put-" + key + "-" + System.currentTimeMillis();

// 策略 2：UUID（简单，但重试时需要保存）
String requestId = UUID.randomUUID().toString();

// 策略 3：业务序号（最严格）
String requestId = "order-" + orderId + "-payment";
```

### 6.2 重试逻辑

```java
public KVResponse putWithRetry(String key, String value, int maxRetries) {
    // 生成 requestId（重试时保持不变）
    String requestId = "put-" + key + "-" + System.currentTimeMillis();
    
    for (int i = 0; i < maxRetries; i++) {
        try {
            KVResponse response = httpClient.put("/kv/" + key, 
                Map.of("value", value, "requestId", requestId));
            
            if (response.isSuccess()) {
                return response;
            }
            
            // 超时或其他错误，重试
            log.warn("PUT failed, retrying... attempt={}", i + 1);
            
        } catch (Exception e) {
            log.error("PUT exception, retrying...", e);
        }
    }
    
    return KVResponse.failure("Max retries exceeded", requestId);
}
```

## 七、注意事项

1. **requestId 必须唯一**：不同业务操作应使用不同的 requestId
2. **重试时使用相同 requestId**：保证幂等性
3. **缓存大小限制**：当前实现最多缓存 10000 个请求，超过后会清空
4. **节点重启后缓存丢失**：生产环境应该持久化 requestId 缓存

# Raft KV Client 实现总结

## 一、客户端概述

已成功为 Raft KV Store 构建一个功能完善的 Java 客户端。

### 核心特性

✅ **幂等性支持**：基于 RequestId 的幂等去重机制  
✅ **自动重试**：内置指数退避重试策略  
✅ **Leader 自动重定向**：自动跟随 Leader 切换  
✅ **线程安全**：可在多线程环境中使用  
✅ **健康检查**：检测服务可用性  
✅ **零外部 HTTP 依赖**：使用 Java 11+ 内置 HttpClient

## 三、核心实现

### 3.1 RaftKVClient.java

**主要功能**：

1. **Builder 模式构造**
```java
RaftKVClient client = RaftKVClient.builder()
        .serverUrls(Arrays.asList("http://localhost:9081"))
        .maxRetries(3)
        .timeoutSeconds(3)
        .build();
```

2. **幂等性支持**
```java
// 自动生成 requestId
public KVResponse put(String key, String value) {
    String requestId = generateRequestId(key, value);
    return put(key, value, requestId);
}

// 指定 requestId
public KVResponse put(String key, String value, String requestId) {
    Map<String, Object> body = new HashMap<>();
    body.put("value", value);
    body.put("requestId", requestId);
    
    return executeWithRetry("PUT", () -> {
        String url = getCurrentLeader() + "/kv/" + key;
        return executeRequest(url, "PUT", body);
    });
}
```

3. **自动重试 + 指数退避**
```java
private KVResponse executeWithRetry(String operation, RequestExecutor executor) {
    Exception lastException = null;

    for (int attempt = 1; attempt <= maxRetries; attempt++) {
        try {
            KVResponse response = executor.execute();

            // 检查是否需要重定向到 Leader
            if (response.isNotLeader() && response.getLeaderUrl() != null) {
                log.info("Redirecting to Leader: {}", response.getLeaderUrl());
                currentLeader.set(response.getLeaderUrl());
                
                // 立即重试
                response = executor.execute();
            }

            return response;

        } catch (Exception e) {
            lastException = e;
            log.warn("{} attempt {} failed: {}", operation, attempt, e.getMessage());

            if (attempt < maxRetries) {
                // 指数退避：100ms, 200ms, 400ms...
                long waitTime = (long) Math.pow(2, attempt - 1) * 100;
                log.debug("Waiting {}ms before retry...", waitTime);
                Thread.sleep(waitTime);
            }
        }
    }

    throw new RuntimeException(operation + " failed after " + maxRetries + " attempts", lastException);
}
```

4. **Leader 自动重定向**
```java
private final AtomicReference<String> currentLeader = new AtomicReference<>();

private String getCurrentLeader() {
    String leader = currentLeader.get();
    if (leader == null || leader.isEmpty()) {
        // 如果没有 Leader，使用第一个节点
        leader = serverUrls.get(0);
        currentLeader.set(leader);
    }
    return leader;
}

// 收到 NOT_LEADER 响应后自动更新
if (response.isNotLeader() && response.getLeaderUrl() != null) {
    currentLeader.set(response.getLeaderUrl());
}
```

5. **HTTP 请求执行**
```java
private KVResponse executeRequest(String url, String method, Map<String, Object> body) throws Exception {
    HttpRequest.Builder requestBuilder = HttpRequest.newBuilder()
            .uri(URI.create(url))
            .timeout(Duration.ofSeconds(timeoutSeconds))
            .header("Content-Type", "application/json");

    if ("PUT".equals(method)) {
        String json = objectMapper.writeValueAsString(body);
        requestBuilder.PUT(HttpRequest.BodyPublishers.ofString(json));
    } else if ("DELETE".equals(method)) {
        requestBuilder.DELETE();
    } else {
        requestBuilder.GET();
    }

    HttpRequest request = requestBuilder.build();
    HttpResponse<String> response = httpClient.send(request, 
            HttpResponse.BodyHandlers.ofString());

    if (response.statusCode() == 200 || response.statusCode() == 301) {
        return objectMapper.readValue(response.body(), KVResponse.class);
    } else {
        throw new RuntimeException("HTTP " + response.statusCode() + ": " + response.body());
    }
}
```

### 3.2 KVResponse.java

**响应实体类**：

```java
@Data
@Builder
public class KVResponse {
    private boolean success;
    private String value;
    private String error;
    private String requestId;
    private String servedBy;
    private String leaderEndpoint;
    private String key;

    // 检查是否是需要重定向到 Leader
    public boolean isNotLeader() {
        return !success && "NOT_LEADER".equals(error);
    }

    // 获取 Leader 地址（带协议前缀）
    public String getLeaderUrl() {
        if (leaderEndpoint == null || leaderEndpoint.isEmpty()) {
            return null;
        }
        if (!leaderEndpoint.startsWith("http://") && !leaderEndpoint.startsWith("https://")) {
            return "http://" + leaderEndpoint;
        }
        return leaderEndpoint;
    }
}
```

## 四、使用方式

### 4.1 基本使用

```java
// 1. 创建客户端
RaftKVClient client = RaftKVClient.builder()
        .serverUrls(Arrays.asList(
                "http://localhost:9081",
                "http://localhost:9082",
                "http://localhost:9083"
        ))
        .maxRetries(3)
        .timeoutSeconds(3)
        .build();

// 2. PUT 操作
KVResponse putResponse = client.put("name", "Alice");
System.out.println("成功: " + putResponse.isSuccess());

// 3. GET 操作
KVResponse getResponse = client.get("name");
System.out.println("name = " + getResponse.getValue());

// 4. DELETE 操作
KVResponse deleteResponse = client.delete("name");
System.out.println("删除成功: " + deleteResponse.isSuccess());
```

### 4.2 幂等性使用

```java
// 生成幂等 requestId
String requestId = "order-123-payment";

// 首次请求
KVResponse response1 = client.put("order:123:status", "paid", requestId);

// 超时重试（使用相同 requestId）
KVResponse response2 = client.put("order:123:status", "paid", requestId);

// 服务端保证幂等：相同 requestId 只执行一次
```

### 4.3 健康检查

```java
boolean healthy = client.healthCheck();
if (healthy) {
    System.out.println("服务正常");
} else {
    System.out.println("服务不可用");
}
```

## 五、工作原理

### 5.1 请求流程

```
1. 客户端发起请求
   ↓
2. 检查当前 Leader 地址
   ↓
3. 发送 HTTP 请求
   ↓
4. 收到响应
   ↓
5. 检查是否需要重定向
   - 是：更新 Leader 地址，立即重试
   - 否：返回响应
   ↓
6. 如果失败，等待后重试（指数退避）
   ↓
7. 成功或超过重试次数
```

### 5.2 重试策略

```
第 1 次失败 → 等待 100ms → 第 2 次
第 2 次失败 → 等待 200ms → 第 3 次
第 3 次失败 → 等待 400ms → 第 4 次
第 4 次失败 → 抛出异常
```

### 5.3 Leader 重定向

```
客户端 → Follower: PUT request
Follower → 客户端: NOT_LEADER + leaderEndpoint
客户端 → Leader: PUT request（自动重定向）
Leader → 客户端: Success
```
